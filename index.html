Section 6.3.2 — Module State Machine (MSM)

① Resetting

Transient state when hardware reset is asserted. All circuitry shuts down. Module exits immediately when reset is deasserted.

② Reset

Ground state. Lowest power consumption. Module retains only essential hardware state. Awaits management interface initialization.

③ MgmtInit

Module initializes the management communication interface (I2CMCI). Minimum memory registers become accessible.

④ ModuleLowPwr

Basic manageability is available. Host can read/write management registers. Data paths are not powered. LowPwrRequest is asserted.

⑤ ModulePwrUp

Module is powering up. Transceiver hardware initializing. Duration controlled by PowerUp timing parameters.

⑥ ModuleReady ★

Fully operational. Data paths can be configured and activated. Normal operating state for all transceiver functions.

⑦ ModulePwrDn

Module powering down. Initiated by host asserting LowPwrRequest. Transitions back to ModuleLowPwr when complete.

⑧ ModuleFault

A module-level fault has occurred. Fault details readable in FaultCause register. Recovery requires hardware reset.

Section 6.3.3 — Data Path State Machine (DPSM) · Per data path instance

① DPDeactivated

Ground state. Data path is inactive, no signal traffic. Host triggers DPInit by writing ApplyDPInit register.

② DPInit

Data path is being initialized with the staged control set configuration. Module processes AppSel and equalization settings.

③ DPInitialized

Data path initialized. Signal integrity settings applied. Ready for Tx activation or deinitialization. Can be reconfigured here.

④ DPTxTurnOn

Tx laser is being turned on. Transient state while laser biasing, modulation, and output power stabilize.

⑤ DPActivated ★

Fully operational. Tx laser is on, Rx path active. Data is flowing. Hot reconfiguration via ApplyImmediate is possible.

⑥ DPDeinit

Data path deinitializing. Laser off, clocks and PLLs powering down. Returns to DPDeactivated when complete.

⑦ DPTxTurnOff

Tx laser being turned off. Used in hot reconfiguration: after turn-off, module returns to DPInitialized for re-configuration.
